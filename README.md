## Автоматическая генерация видео о рыбалке для YouTube (GitHub Actions)

Этот проект генерирует **короткие вертикальные ролики (Shorts)** для YouTube‑канала о рыбалке **полностью в GitHub Actions**, без зависимости от локальной машины.

Основной сценарий:

- GitHub Actions по расписанию или вручную запускает workflow `Generate Fishing Short`.
- Скрипт `generate_fishing_short.py`:
  - получает сценарий через LLM (Groq API) или использует запасной;
  - скачивает вертикальные клипы с Pexels (по `PEXELS_API_KEY`);
  - генерирует озвучку на русском через gTTS;
  - скачивает бесплатную фоновую музыку;
  - собирает вертикальное видео 1080×1920 с динамично меняющимися кадрами и субтитрами оптимальной длительности.

Итоговый файл `build/output_fishing_short.mp4` прикладывается к запуску workflow как artifact.

### 1. Настройка репозитория на GitHub

1. Создайте репозиторий и загрузите в него содержимое папки проекта.
2. В разделе `Settings → Secrets and variables → Actions` добавьте секреты:
   - `GROQ_API_KEY` — ключ к Groq LLM API (для генерации сценария);
   - `PEXELS_API_KEY` — ключ к Pexels API (для скачивания видео по запросу `fishing`).
3. Убедитесь, что в репозитории есть файлы:
   - `generate_fishing_short.py`
   - `.github/workflows/generate_fishing_short.yml`
   - `requirements.txt`

### 2. GitHub Actions workflow

Workflow описан в `.github/workflows/generate_fishing_short.yml` и делает следующее:

- запускается по расписанию (`cron`) и по кнопке (`workflow_dispatch`);
- ставит Python 3.11 и `ffmpeg` на хосте Ubuntu;
- устанавливает зависимости из `requirements.txt`;
- запускает `python generate_fishing_short.py`;
- загружает результат `build/output_fishing_short.mp4` как artifact `fishing-short`.

После первого пуша вы можете зайти на вкладку **Actions**, выбрать `Generate Fishing Short` и выполнить **Run workflow**, чтобы получить тестовый ролик.

### 3. Как работает `generate_fishing_short.py`

- **Сценарий** (`call_groq_for_script`):
  - если задан `GROQ_API_KEY`, вызывается Groq Chat Completions API с промптом сценариста про рыбалку;
  - ожидается JSON с массивом частей `parts[]`, каждая часть — фраза для отдельного кадра;
  - если что‑то пошло не так, используется запасной сценарий про ошибки при ловле щуки.

- **Футаж** (`download_pexels_clips`):
  - по `PEXELS_API_KEY` вызывается `https://api.pexels.com/videos/search?query=fishing&orientation=portrait`;
  - скачиваются несколько вертикальных видео‑клипов, сохраняются в `build/clips/`;
  - далее для каждого кадра выбирается случайный фрагмент клипа для живой картинки.

- **Озвучка** (`build_tts_audio`):
  - все фразы сценария склеиваются в один текст;
  - через gTTS создаётся `build/voice.mp3` на русском.

- **Музыка** (`download_background_music`):
  - при первом запуске скачивается один трек из набора бесплатных ссылок (Free Music Archive);
  - сохраняется как `build/music.mp3` и используется в последующих запусках, если не удалять.

- **Сборка видео** (`build_video`):
  - для каждой фразы считается комфортная длительность кадра по длине текста (примерно 15 символов/сек, в диапазоне 1.8–3.8 сек);
  - все длительности масштабируются под фактическую длину озвучки, чтобы голос и кадры совпадали;
  - для каждого кадра:
    - выбирается случайный отрезок в одном из Pexels‑клипов;
    - клип приводится к вертикали 1080×1920;
    - поверх рисуется субтитр снизу безопасного размера;
  - клипы склеиваются в один ролик;
  - аудио: голос + приглушённая фонова музыка накладываются через `CompositeAudioClip`;
  - результат записывается в `build/output_fishing_short.mp4` (H.264 + AAC, 30 fps).

### 4. Локальный тест

Для отладки можно запустить генератор локально (на Windows или Linux):

1. Установите Python 3.11+ и `ffmpeg` (чтобы он был в `PATH`).
2. В корне проекта выполните:
   ```bash
   pip install -r requirements.txt
   ```
3. Экспортируйте переменные окружения (или задайте их в терминале):
   ```bash
   export GROQ_API_KEY=...
   export PEXELS_API_KEY=...
   ```
4. Запустите:
   ```bash
   python generate_fishing_short.py
   ```
5. Готовое видео появится в `build/output_fishing_short.mp4`.
