## Воркфлоу загрузки на YouTube в n8n

Этот файл описывает, как организовать отдельный воркфлоу в n8n для загрузки уже сгенерированного ролика и метаданных на YouTube после ручного ревью.

### 1. Предпосылки

Основной воркфлоу `YouTube Fishing Shorts Generator`:

- сохраняет сгенерированное видео в папку `output/` (файл `output_shorts.mp4`);
- сохраняет метаданные в `runtime/video_meta.json`;
- останавливается на ноде **Manual Review (Stop Here)**, чтобы вы могли:
  - просмотреть видео;
  - при необходимости поправить `runtime/video_meta.json` вручную;
  - принять решение о публикации.

### 2. Отдельный воркфлоу загрузки (Upload Workflow)

Создайте новый воркфлоу в n8n, например `YouTube Fishing Upload`, со следующими шагами:

1. **Manual Trigger / Webhook / Telegram** — точка входа:
   - либо полностью ручной запуск через **Manual Trigger**;
   - либо запуск по ссылке (Webhooks) или через бота (Telegram), если хотите триггерить из внешнего приложения.

2. **Read Meta JSON (Read Binary File / Function + File System)**  
   - прочитать файл `runtime/video_meta.json`;
   - распарсить JSON, чтобы получить `title`, `description`, `tags`.

   Простейший вариант через ноду **Function**:

   - использовать Node.js‑код для чтения файла с диска (через встроенные способности ноды `Function` или через отдельный HTTP/FS‑сервис, если это ограничено вашей установкой n8n);
   - результат — объект:
     - `meta.title`
     - `meta.description`
     - `meta.tags[]`.

3. **Locate Video File**  
   - определить путь к сгенерированному видео, по умолчанию: `output/output_shorts.mp4`;
   - при желании можно хранить имя файла в отдельном JSON рядом с метаданными.

4. **YouTube → OAuth2 Credentials**  
   - в разделе **Credentials** n8n создайте подключение **Google OAuth2 API** с доступом к YouTube Data API;
   - при первом использовании n8n попросит авторизацию в вашем аккаунте Google (канале YouTube);
   - сохраните эти креденшлы и используйте их в ноде загрузки.

5. **Upload Video (HTTP Request к YouTube Data API или готовая нода, если доступна)**  
   Вариант с HTTP‑нодой:

   - нода **HTTP Request**:
     - `Method`: `POST`;
     - `URL`: `https://www.googleapis.com/upload/youtube/v3/videos?uploadType=resumable&part=snippet,status`;
     - тело запроса (JSON) содержит:
       - `snippet.title` — `meta.title`;
       - `snippet.description` — `meta.description`;
       - `snippet.tags` — `meta.tags`;
       - `snippet.categoryId` — ID категории (например, `17` — Sports, или другая подходящая);
       - `status.privacyStatus` — `private` / `unlisted` / `public`;
       - при планировании публикации: поле `status.publishAt` (ISO‑дата/время).

   - загрузка файла может выполняться:
     - либо по протоколу resumable upload (два запроса: инициализация + отправка бинарного тела);
     - либо через готовую YouTube‑нод, если она есть в вашей версии n8n (в этом случае настройте её в UI, указав путь к файлу и метаданные).

   Для упрощённой настройки рекомендуется:

   - проверить, есть ли в вашей установке готовый узел **YouTube**;
   - если есть — использовать его и передать:
     - путь к видеофайлу;
     - `title`, `description`, `tags`, `privacyStatus` и, при необходимости, `publishAt`.

6. **Update Google Sheets Status (Optional)**  
   - после успешной загрузки обновить строку в Google Sheets:
     - `status` → `done`;
     - добавить ссылку на опубликованный ролик (поле `url`), если вы храните его в таблице.

### 3. Ручной ревью и планирование публикаций

Чтобы ревью и планирование публикаций были удобными:

- оставляйте воркфлоу генерации видео только до шага сохранения файлы + метаданных;
- после проверки:
  - при необходимости правьте `runtime/video_meta.json` вручную (заголовок, описание, теги);
  - запускайте воркфлоу `YouTube Fishing Upload` вручную и указывайте:
    - какой файл видео использовать;
    - какой JSON метаданных взять;
    - какой выбрать `privacyStatus` и `publishAt`.

Так вы получаете:

- полностью автоматизированную техническую часть (сценарий → озвучка → сборка видео);
- контроль за финальной публикацией и расписанием (решение остаётся за вами).
